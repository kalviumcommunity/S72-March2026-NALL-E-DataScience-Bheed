# Milestone 4.8 — Understanding Notebook Cells: Code vs Markdown

## Overview
As part of our commitment to high-quality project documentation, we have created `notebooks/02_cells_demo.ipynb` to establish a standard for how code and explanations should be combined in our future analyses.

## Key Learnings
1.  **Code Cells**: Used for data manipulation, calculations, and plotting. In the demo, we calculated the average monthly footfall for Jaipur.
2.  **Markdown Cells**: Essential for telling the "story" behind the data. We used:
    *   **Headers** to structure sections.
    *   **Bold/Italic** for emphasis on key metrics.
    *   **LaTeX** for mathematical clarity ($\bar{x} = \frac{\sum x_i}{n}$).
    *   **Lists** to summarize the "Mini-EDA" findings.

## Sample Findings (Mini-EDA)
Using the sample data, we identified a clear increase in footfall during the holiday season (Oct-Dec) and a significant dip during the peak summer month (June).
