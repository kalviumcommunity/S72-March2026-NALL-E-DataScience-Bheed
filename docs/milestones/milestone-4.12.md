# Milestone 4.12 — Organising Raw Data, Processed Data, and Output Artifacts

## Data Folder Conventions
To ensure reproducibility and clarity, we have established the following conventions for our data folders:

1.  **`data/raw/`**:
    *   **Purpose**: Stores original, unaltered source data.
    *   **Rule**: Never edit these files directly. All processing must happen via scripts in `src/`.
    *   **File Created**: `sample_footfall.csv` (1000 rows of synthetic tourism data).

2.  **`data/processed/`**:
    *   **Purpose**: Stores cleaned, transformed, and engineered data ready for modeling.
    *   **Rule**: These files are generated by `src/data_pipeline.py`.
    *   **Placeholder**: `.gitkeep` ensures the directory is tracked.

3.  **`outputs/`**:
    *   **Purpose**: Stores artifacts generated by the analysis.
    *   **`eda_charts/`**: Location for PNG/PDF plots from the EDA notebook.
    *   **`data_summary.md`**: Detailed schema documentation (created below).

## Files Created
*   **`data/raw/sample_footfall.csv`**: Realistic synthetic data for 5 cities (Jaipur, Goa, Shimla, Udaipur, Manali) spanning 50 discrete time periods.
*   **`outputs/data_summary.md`**: Dataset schema and descriptions.
